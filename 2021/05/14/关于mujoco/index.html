mujoco环境render时报错

GLEW initalization error: Missing GL version

openai/mujoco-py issue

Step 1
Run update command to update package repositories and get the latest package information.
sudo apt-get update -y
Step 2
Run the install command with -y

sudo apt-get install -y libglew-dev
Step 3:
export LD_PRELOAD=/usr/lib/x86_64-linux-gnu/libGLEW.so

export设置只在当前的bash登录的session有效

使用Monitor error: missing GL version

在不使用Monitor，单纯使用mujoco环境时，增加export LD_PRELOA=/...命令有效，但是使用Monitor后仍然报错Missing GL version ...

这个问题通过增加/usr/lib/x86_64-linux-gnu同目录的libGL.so

命令locate ...

修复了该问题后，录像仍然存在问题，mp4文件打不开，并且文件大小存在问题，只有262bytesgithub issues

gym该bug修复，是库代码中的一个缩进失误，将303行的self.proc.stdin.write(frame.tobytes())从if条件中移出并保存，解决该问题。

该issue于2020年12月提出并修正，但是在2021年5月，安装gym1.8版本后，仍然存在该问题，可以手动修复
